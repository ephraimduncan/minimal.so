# Changelog Page Redesign

## Context

Current changelog is a plain vertical list with prose styling. Redesign adopts the two-column grid layout from the reference UI (date left, content right) while keeping it appropriate for a versioned changelog with categorized entries.

## Changes

Single file: `app/changelog/page.tsx`

### Layout: Two-column grid

Shared grid pattern for header + each version article:
```
grid sm:grid-cols-[140px_1fr] sm:gap-6 md:grid-cols-[180px_1fr]
```

- **Header**: empty left col, right col has back link → h1 → subtitle
- **Per version**: left col = date (right-aligned on sm+), right col = version title + categorized entries
- Versions separated by `gap-12`
- Remove `prose` wrapper — use explicit spacing instead

### Category badges

Replace plain "Added"/"Changed"/"Fixed" h3 headers with `Badge` component using semantic color tokens:

| Category | Badge style |
|----------|------------|
| Added | `bg-success/10 text-success-foreground` |
| Changed | `bg-info/10 text-info-foreground` |
| Fixed | `bg-destructive/10 text-destructive-foreground` |

### Other changes

- Replace inline SVG arrow with `IconChevronLeft` from `@tabler/icons-react`
- Wrap dates in `<time>` elements with `dateTime` attribute
- Add `id="v{version}"` anchors to each version section
- No bullet points on entries — clean flat list with spacing (matches reference UI)
- Version titles are anchor links (`<a href="#v0.1.0">v0.1.0</a>`) with hover underline
- PR links: `text-muted-foreground hover:text-foreground text-xs font-mono`

### Reused existing code

- `Badge` from `components/ui/badge.tsx`
- `IconChevronLeft` from `@tabler/icons-react`
- Semantic color tokens from `globals.css` (success, info, destructive) — no CSS changes needed
- `cn()` from `lib/utils.ts`
